Duplicated Code

Description

The same code structure — or near-identical logic — appears in more than one place. It might be a copy-pasted method, two classes with similar algorithms that diverged slightly, or the same conditional pattern repeated across unrelated files. Every duplication is a ticking liability: when the logic needs to change, you have to find and update every copy. Miss one and you have a bug that works correctly almost everywhere.

Duplication isn't always literal. Two blocks of code can look different on the surface but express the same intent — different variable names, slightly reordered statements, one using a for loop and the other a while. These semantic duplicates are harder to spot but just as dangerous.

This is arguably the most fundamental smell. Many other smells (Data Clumps, Shotgun Surgery, Parallel Inheritance Hierarchies) are just specific patterns of duplication. If you could only fix one smell for the rest of your career, fix this one.

Example

public class OrderService {

    public double calculateOrderTotal(Order order) {
        double total = 0;
        for (OrderItem item : order.getItems()) {
            double linePrice = item.getPrice() * item.getQuantity();
            if (item.getCategory().equals("electronics")) {
                linePrice *= 0.95;
            } else if (item.getCategory().equals("books")) {
                linePrice *= 0.90;
            }
            total += linePrice;
        }
        if (order.getCustomer().isVip()) {
            total *= 0.90;
        }
        return total;
    }
}

public class InvoiceService {

    public String generateInvoice(Order order) {
        // same calculation, slightly different variable names
        double subtotal = 0;
        for (OrderItem item : order.getItems()) {
            double price = item.getPrice() * item.getQuantity();
            if (item.getCategory().equals("electronics")) {
                price *= 0.95;
            } else if (item.getCategory().equals("books")) {
                price *= 0.90;
            }
            subtotal += price;
        }
        if (order.getCustomer().isVip()) {
            subtotal *= 0.90;
        }
        return "Invoice total: $" + String.format("%.2f", subtotal);
    }
}

public class ReportService {

    public Map<String, Double> getOrderSummary(List<Order> orders) {
        Map<String, Double> summary = new HashMap<>();
        for (Order order : orders) {
            // and again...
            double total = 0;
            for (OrderItem item : order.getItems()) {
                double lp = item.getPrice() * item.getQuantity();
                if (item.getCategory().equals("electronics")) {
                    lp *= 0.95;
                } else if (item.getCategory().equals("books")) {
                    lp *= 0.90;
                }
                total += lp;
            }
            if (order.getCustomer().isVip()) {
                total *= 0.90;
            }
            summary.put(order.getId(), total);
        }
        return summary;
    }
}

The pricing logic — category discounts and VIP discount — is duplicated three times across three services. When the electronics discount changes from 5% to 8%, you update three places. When a "clothing" category is added, you update three places. If the VIP threshold becomes tiered, you update three places.

What principle does it break?

DRY (Don't Repeat Yourself). Every piece of knowledge should have a single, authoritative representation. Duplicated code means duplicated knowledge — the business rule "electronics get a 5% discount" exists in three places, with no way to enforce that they stay in sync.

How to fix it

Extract Method — same class

When the duplication is within one class, extract the shared logic into a method.

private double calculateLinePrice(OrderItem item) {
    double price = item.getPrice() * item.getQuantity();
    if (item.getCategory().equals("electronics")) return price * 0.95;
    if (item.getCategory().equals("books")) return price * 0.90;
    return price;
}

Extract Class — across classes

When the duplication spans multiple classes, consolidate the logic into a single class that all callers share.

public class OrderPricingCalculator {

    public double calculateTotal(Order order) {
        double total = order.getItems().stream()
            .mapToDouble(this::calculateLinePrice)
            .sum();
        if (order.getCustomer().isVip()) {
            total *= 0.90;
        }
        return total;
    }

    private double calculateLinePrice(OrderItem item) {
        double price = item.getPrice() * item.getQuantity();
        if (item.getCategory().equals("electronics")) return price * 0.95;
        if (item.getCategory().equals("books")) return price * 0.90;
        return price;
    }
}

All three services now delegate:

public class OrderService {
    private final OrderPricingCalculator pricing;

    public double calculateOrderTotal(Order order) {
        return pricing.calculateTotal(order);
    }
}

public class InvoiceService {
    private final OrderPricingCalculator pricing;

    public String generateInvoice(Order order) {
        double total = pricing.calculateTotal(order);
        return "Invoice total: $" + String.format("%.2f", total);
    }
}

public class ReportService {
    private final OrderPricingCalculator pricing;

    public Map<String, Double> getOrderSummary(List<Order> orders) {
        Map<String, Double> summary = new HashMap<>();
        for (Order order : orders) {
            summary.put(order.getId(), pricing.calculateTotal(order));
        }
        return summary;
    }
}

One pricing rule, one place. Change it once, it's correct everywhere.

Pull Up Method — sibling subclasses

When two subclasses contain identical methods, move the method to the parent.

Form Template Method

When two subclasses have methods with the same structure but different details, define the structure in the parent and let each subclass override the varying steps.
