Code Smells

Complete Reference

View Refactoring Techniques →

1 — Bloaters

Code, methods, and classes that have grown excessively large and hard to work with.

  • Long Method
  • Large Class
  • Primitive Obsession
  • Long Parameter List
  • Data Clumps
  • Combinatorial Explosion
  • Global Data
  • Mutable Data
  • Loops

2 — Object-Orientation Abusers

Cases where object-oriented principles are applied incompletely or incorrectly.

  • Switch Statements
  • Alternative Classes with Different Interfaces
  • Temporary Field
  • Refused Bequest
  • Conditional Complexity

3 — Change Preventers

Smells that make changing one thing require changing many other places.

  • Divergent Change
  • Shotgun Surgery
  • Solution Sprawl
  • Parallel Inheritance Hierarchies
  • Side Effect

4 — Dispensables

Things that are unnecessary or unclear, whose removal or improvement makes code cleaner.

  • Duplicated Code
  • Lazy Class
  • Speculative Generality
  • Comments (Deodorant Comment)
  • Data Class
  • Dead Code
  • Mysterious Name / Inefficient Name
  • Oddball Solution / Black Sheep
  • Lost Intent

5 — Couplers

Smells that cause excessive coupling between classes.

  • Inappropriate Intimacy
  • Feature Envy
  • Message Chains
  • Middle Man
  • Indecent Exposure
  • Incomplete Library Class
